Returning

Smalltalk has these implicit return values:
- Objects 'return' themselves.
- Blocks return the result of last expression in them.
- Methods return the current object self by default.
  (So not 'null', 'undefined' or 'void' as in other languages)

You can explicitly return a value with the caret character: ^.
An explicit return immediately exits the current method.
Even if the return is within a (nested) block it wil still exit the current method.

Examples:
 ^ 'a'.
'b'.
 This will return 'a' since 'b' is never reached.

1 < 2 ifTrue: [ ^ 'smaller' ].
^ 'bigger'
 This will return 'smaller' since 'bigger' is never reached.
