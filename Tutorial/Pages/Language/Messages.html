Messages

Smalltalk objects communicate by sending messages to each other.
This is equivalent to calling 'functions' or 'methods' in other languages.

Sending a message in Smalltalk is done by simply placing it after an object.
Example: Evaluating 4 squared in the Playground will give 16.
This sends the message squared to the object 4.
Then the squared method in the Integer class
multiplies itself with itself and returns the resulting Integer object.

Message types

Smalltalk has three message types:

1 - Unary messages

The 'unary' message type has no arguments (parameters)
it's name starts with a lower case letter.

Examples:
 - 4 squared returns 16.
- 'ABC' toLowerCase returns 'abc'.
- true not returns false.

2 - Binary messages

The 'binary' message type has one argument
it's name consists of one or more symbols.
Binary messages mostly implement numerical or logical operations.

Examples:
 - 2 + 3 returns 5 (addition).
- 'abc' , 'def' returns 'abcdef' (concatenation).
- true & false returns false (logic and).

3 - Keyword messages

The 'keyword' message type has one or more arguments
where each argument is preceded by a keyword.
A keyword starts with a lower case letter and end with colon.

Examples:
 - 6 gcd: 9 returns 3 (greatest common denominator).
- 'abc' replace: 'b' with: '2' returns 'a2c' (find/replace).

Message evaluation order

Left to right evaluation

Message sends of the same type are evaluated left-to-right.

Examples:
 - 4 squared negated returns -16 (first square, then negate).
- But 4 negated squared returns 16 (because -4 * -4 = 16).
- 1 + 2 * 3 returns 9 (not 7, left-to-right evaluation).
- ( 'abcd' substring: 2 ) repeat: 2 returns 'cdcd'.
  (subsequent keyword messages need parentheses to separate them)

Unary, binary, keyword evaluation

In expressions with different message types, unary messages are evaluated first,
then binary messages and lastly keyword messages.

Example:
 - 9 gcd: 10 - 2 squared returns 3.

The evaluation order is:
First the unary message 2 squared is evaluated, returning 4.
Then the binary message 10 - 4 is evaluated, returning 6.
Lastly the keyword message 9 gcd: 6 is evaluated, returning 3.

Cascading messages (advanced)

Sometimes multiple messages need to be sent to the same object.
In Smalltalk, this can be done by separating the messages with a semicolon.

Example:
 - Person new
  firstName: 'Jane' ;
  lastName: 'Doe'

The evaluation order is:
First Person new creates a new person object.
Then firstName: 'Jane' sets the first name on that person.
Lastly lastName: 'Doe' sets the sets the last on that same person.
